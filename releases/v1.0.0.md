# Little_Shijimi v1.0.0

Released: `2026-04-09`

## Summary

`v1.0.0` promotes Little_Shijimi from a Telegram notifier into a practical remote approval bridge for Codex. The wrapper can now register approval prompts per terminal, accept `go` and `no` replies from Telegram, and route the corresponding input back into the correct terminal session.

## Highlights

- Added `telegram_reply_listener.py` to poll Telegram replies and dispatch approval actions to pending prompts.
- Added terminal-scoped pending approval records so replies target the right Codex session.
- Added menu-aware approval input derivation so the wrapper can choose `Enter` or arrow-key navigation based on the rendered approval options.
- Added `telegram_approval_replay_tester.py` for offline end-to-end validation using a historical prompt and mocked Telegram responses.
- Refactored Telegram helper logic to support reusable API requests and mocked local test fixtures.
- Updated installation and README guidance to reflect the symlink-based local install path.

## Verification

- Offline replay tester exercises the approval registration, reply polling, and terminal dispatch path without using a live session.

## Upgrade Notes

- Re-run `./install.sh` if your local `~/.local/bin/codex` is not already a symlink to this repo's `wrappers/codex`.
- If your approval UI does not start on the first visible option, set `AGENT_NOTIFY_APPROVAL_START_INDEX` to match your local focus position.
